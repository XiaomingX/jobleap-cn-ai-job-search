## 💡 学长/学姐的修改建议
### 1. 拒绝只有“苦劳”没有“功劳”
❌ **Before：**
“参与了电商系统的开发，负责订单模块。”（谁知道你干了啥？）

✅ **After：**
“基于 Spring Cloud 重构订单模块，引入 RabbitMQ 实现削峰填谷，将订单处理峰值 TPS 从 500 提升至 2000+。”（这就叫专业！）

### 2. 技术栈要写得“有心机”
别列一堆你只会写 Hello World 的技术。要按照 **精通 > 熟悉 > 了解** 的顺序排。重点突出的 Java 基础、并发编程和数据库能力。
