# 程序员技术简历撰写指南

程序员简历应当体现“专业感”和“解决问题的能力”。一份能打的技术简历不仅要有深厚的技术栈，更要有清晰的项目逻辑。

## 一、技术栈的正确写法

- **分类列出**：按语言、框架、数据库、工具/中间件分类。
- **准确拼写**：注意大小写（如 MySQL, Python, Redis）。
- **体现程度**：明确区分“精通”、“熟悉”与“了解”。

## 二、项目经历：简历的灵魂

### STAR 法则应用
- **S (Situation)**：项目背景是什么？
- **T (Task)**：你负责的具体任务是什么？
- **A (Action)**：你用了什么技术、解决了什么难点？
- **R (Result)**：最终产出了什么价值（吞吐量提升、延迟降低、代码量减少等）。

### 关注点
- 重点写“你做了什么”，而非“项目做了什么”。
- 避免硬编经验，必须能扛住面试官的深度追问。

## 三、加分项与亮点

- **GitHub/开源贡献**：如果你有认真维护的项目或对知名库提过 PR。
- **技术博客**：展现你的思考深度和总结能力。
- **专业证书**：如 CKA, AWS 认证等具有公信力的证书。
- **算法竞赛**：如 ACM, 蓝桥杯奖项。
