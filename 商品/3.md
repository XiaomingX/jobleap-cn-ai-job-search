在中国，从事存储芯片、内存系统相关编程（即底层驱动、固件、控制器开发和系统优化）的公司绝对不属于游戏公司，它们主要分布在 集成电路（IC/半导体）、服务器/企业级硬件、云计算/互联网巨头、以及基础软件 这四大类别中。

游戏公司（如腾讯、网易、米哈游）做的是应用层和引擎层的内存管理（例如用 C++ 写内存池、优化内存对齐以防止缓存失效），使用的是操作系统提供的通用 API，并不直接和存储芯片物理硬件打交道。

以下是真正进行存储芯片相关编程的四大类中国公司及其工作内容：

1. 集成电路设计与存储芯片厂商 (IC / Semis)
这是最核心、最直接对存储芯片进行编程的类别。他们设计芯片，并为其编写运行在芯片内部的固件（Firmware）或驱动。

代表企业：
存储介质与设计龙头：长江存储（YMTC，做 NAND 闪存）、长鑫存储（CXMT，做 DRAM 内存）、澜起科技（Montage，做 DDR5 接口芯片和 CXL 内存扩展控制器）。
SSD 主控芯片与固件商：联芸科技、英韧科技、忆恒创源（Memblaze）、得一微电子、忆芯科技。
编程内容：
FTL (Flash Translation Layer，闪存翻译层) 开发：这是 SSD 固件的核心，用 C 语言编写，负责坏块管理、磨损均衡（Wear Leveling）和垃圾回收。
主控 SoC 固件：编写运行在 ARM/RISC-V 架构主控芯片上的实时操作系统（RTOS）代码，实现 PCIe/NVMe 协议栈。
寄存器与协议控制：针对 DDR/CXL 接口控制器的底层物理层（PHY）微代码（Microcode）开发。
2. 服务器与企业级硬件厂商 (Server & Enterprise Hardware)
这类公司购买海力士、美光、三星的内存颗粒以及上述主控芯片，组装成服务器、存储阵列或 CXL 内存池设备。

代表企业： 华为、浪潮信息、新华三（H3C）、中科曙光、联想。
编程内容：
BIOS / UEFI 开发：编写硬件初始化代码，负责在开机时检测和配置内存频率、通道和拓扑结构。
CXL 内存池化管理：编写运行在服务器管理芯片（BMC）上的软件，动态分配 CXL 扩展内存。
存储系统研发：开发企业级存储服务器的 RAID 控制算法和双活高可用固件。
3. 云计算与互联网巨头 (Cloud & Internet Giants)
这些公司是海力士和美光芯片的最大买家。为了极致的系统性能，他们会招募底层开发人员，针对特定硬件进行定制化开发。

代表企业： 阿里（平头哥半导体）、腾讯、字节跳动、百度。
编程内容：
定制化 SSD 固件：由于标准 SSD 在高并发下会出现延迟抖动，大厂会与芯片厂合作，由大厂的软件工程师修改 SSD 固件（Open-Channel SSD/ZNS），让操作系统直接管理闪存物理介质。
内核级内存优化：修改 Linux 内核的内存分配器、多物理节点（NUMA）调度算法，编写 CXL 动态容量（DCD）驱动。
SPDK/PMDK 应用：在自研分布式数据库和缓存系统（如 Redis 优化版）中，使用 SPDK 绕过内核直接读写高速存储。
4. 基础软件与分布式存储公司 (System Software)
专注于开发数据管理软件，让上层应用能以更高效率读写存储芯片。

代表企业： 星辰天合（XSKY）、焱融科技（分布式文件系统）、PingCAP（TiDB 分布式数据库）。
编程内容：
用户态文件系统：使用 C++/Go 编写绕过操作系统的存储驱动，直接与 NVMe 控制器通信，榨干硬件的 IOPS 极限。
键值存储引擎（KV Store）：如针对 NAND 闪存特性优化写入放大的存储引擎（例如 RocksDB 优化）。
