# PHP 后端开发实战指南

## 核心趋势与能力要求
PHP 正在向高性能与常驻进程架构演进。现代 PHP 开发不再局限于简单的 CRUD，而是更看重对协程、微服务及高并发场景的处理能力。

### 1. 技术栈建议
- 核心语言：PHP 8.2/8.3 (精通新特性)。
- 核心框架：Laravel 11, Hyperf (Swoole/Swow)。
- 数据库：MySQL (优化与分库分表), Redis。
- 进阶技术：Swoole, RPC, Docker 容器化。

### 2. 简历避坑与提升
- 避免陈述：熟练使用 TP/Laravel 编写接口。
- 建议陈述：基于 Hyperf 协程框架重构核心服务，将单机并发能力提升 5 倍，接口平均响应时间降低至 50ms。
- 核心词汇：协程 (Coroutine), 异步 I/O, 多级缓存, 容器编排, 服务治理。

### 项目经历

#### Swoole 高并发实时通讯系统
- 情况：PHP-FPM 模式无法高效维持长连接，实时性差。
- 任务：搭建支持高并发的 WebSocket 聊天服务。
- 行动：
    - 采用 Hyperf 协程框架，实现异步非阻塞 I/O 处理。
    - 利用 Redis 实现用户状态管理与消息路由，降低内存消耗。
    - 引入消息队列对高频消息进行削峰，保护核心数据库。
- 结果：单机支持 10w+ 在线连接，消息投递延迟低于 100ms。
