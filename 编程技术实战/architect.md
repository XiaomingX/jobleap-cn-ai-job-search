# 系统架构师实战指南

## 核心职责与素质
架构师需要具备宏观的技术视野与微观的落地能力。简历应重点突出技术决策、复杂系统演进、高可用建设以及对商业目标的支撑能力。

### 1. 核心关注点
- 分布式系统：CAP/BASE 理论应用，异地多活方案。
- 云原生：K8s 编排，Service Mesh 治理，无服务器架构。
- 领域驱动：DDD 实践，服务边界划分，模型建模。
- AI 集成：大模型推理接入，向量数据库应用。

### 2. 简历提升建议
- 避免陈述：精通 Spring Cloud, K8s 等工具。
- 建议陈述：主导核心系统微服务化重构，基于 DDD 拆分服务边界，设计落地异地多活方案，支撑 10w+ TPS 交易峰值。
- 核心词汇：全链路压测, 容灾演练, 分布式事务解决, 技术领导力。

## 简历模板（核心片段）

### 核心竞争力
- 架构演进：深厚的分布式系统架构设计功底，主导过从单体到微服务的完整转型。
- 基础设施：精通 K8s 生态，熟悉容器编排与 CI/CD 自动化流水线。
- 数据一致性：深入理解各种分布式一致性协议，在复杂场景下保证数据可靠性。
- 团队赋能：具备技术前瞻性，能够制定技术规范并带领团队攻克核心技术难题。

### 重点项目经历

#### 金融级分布式交易中台
- 职责：主导系统从 Oracle 单体架构向 Spring Cloud + MySQL 分库分表的分布式架构演进。
- 行动：
    - 设计并实施全链路压测方案，识别并优化 20 余处性能瓶颈。
    - 搭建三级缓存体系，将系统可用性提升至 99.99%。
    - 基于 TCC 模式解决跨服务的最终一致性问题，确保资金安全。
- 结果：平稳支撑高强度流量峰值，系统抗压能力提升数十倍。
