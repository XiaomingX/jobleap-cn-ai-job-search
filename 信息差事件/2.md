**Alembic** 是 Python 中一个轻量级的**数据库迁移（Migration）工具**，由 SQLAlchemy 的作者 Mike Bayer 开发。它专门用于管理关系型数据库的** schema（表结构）变更**，类似于 Git 用于代码版本控制，但针对数据库。

它允许你通过可版本化的 Python 脚本（migration scripts）来：

- 创建/修改/删除表、列、索引、外键等。
- 跟踪数据库 schema 的历史版本。
- 支持升级（upgrade）和降级（downgrade）。
- 自动生成迁移脚本（autogenerate），对比 SQLAlchemy 模型和实际数据库的差异。

### 为什么需要 Alembic？
在开发中，模型经常变化（如新增字段、修改类型、加索引）。手动执行 SQL 容易出错、不易回滚、多人协作困难。Alembic 把这些变更变成**可重现、可版本控制的脚本**，适合团队开发、CI/CD、生产环境部署。

**常见使用场景**：
- Web 项目（FastAPI、Flask + SQLAlchemy）。
- 需要频繁迭代 schema 的后端服务。
- 多环境（dev/test/prod）保持数据库结构一致。
- 生产环境零停机 schema 变更。
- 数据迁移（Data Migration）辅助（如批量更新数据，通常结合 schema 变更）。

### 如何安装和基本使用

1. **安装**：
   ```bash
   pip install alembic sqlalchemy
   # 或 uv/poetry 等工具
   ```

2. **初始化迁移环境**（项目根目录执行一次）：
   ```bash
   alembic init alembic   # 创建 alembic/ 目录和 alembic.ini
   # 对于异步（如 Async SQLAlchemy），用：alembic init -t async migrations
   ```

   生成的结构：
   - `alembic.ini`：配置文件（数据库连接等）。
   - `alembic/env.py`：环境脚本（导入模型、配置 target_metadata）。
   - `alembic/versions/`：存放迁移脚本。
   - `script.py.mako`：迁移脚本模板。

3. **配置**（关键步骤）：
   - 编辑 `alembic.ini`，设置数据库 URL：
     ```
     sqlalchemy.url = postgresql+psycopg2://user:pass@localhost/dbname
     # 或 sqlite:///mydb.db
     ```
   - 编辑 `alembic/env.py`，导入你的 SQLAlchemy 模型，并设置：
     ```python
     from myapp.models import Base  # 你的 Base = declarative_base()
     target_metadata = Base.metadata
     ```

4. **生成迁移脚本**：
   - **自动生成**（推荐，大部分 schema 变更）：
     ```bash
     alembic revision --autogenerate -m "add user table"  # 或 "描述变更"
     ```
     这会对比模型和数据库，生成 `versions/xxxx_add_user_table.py` 文件。

   - **手动创建**（复杂自定义逻辑）：
     ```bash
     alembic revision -m "manual migration"
     ```

5. **应用迁移**：
   ```bash
   alembic upgrade head     # 升级到最新版本
   alembic downgrade -1     # 降级一步
   alembic upgrade <revision_id>  # 升级到指定版本
   ```

6. **常用命令**：
   - `alembic current`：查看当前版本。
   - `alembic history`：查看迁移历史。
   - `alembic stamp head`：标记当前为 head（不执行迁移）。
   - `alembic merge heads`：处理分支合并冲突（多人协作时常见）。

### 迁移脚本示例（upgrade/downgrade）
```python
from alembic import op
import sqlalchemy as sa

# revision identifiers
revision = 'xxxx'
down_revision = None
branch_labels = None
depends_on = None

def upgrade():
    op.create_table('users',
        sa.Column('id', sa.Integer(), primary_key=True),
        sa.Column('name', sa.String(50), nullable=False),
    )
    # op.add_column(...), op.create_index(...) 等

def downgrade():
    op.drop_table('users')
```

### 注意事项和最佳实践
- **Autogenerate** 很强大，但复杂变更（如数据迁移、ENUM 类型、触发器）常需手动补充。
- **生产环境**：先在 staging 测试，考虑零停机（如用 `op.execute()` 执行安全 SQL）。
- **多人协作**：像 Git 一样处理 merge conflicts。
- **数据迁移**：Alembic 主要做 schema，可在 migration 脚本中用 `op.bulk_insert()` 或 ORM 操作迁移数据。
- 与 Django/Flask-Migrate 类似，但更灵活（纯 SQLAlchemy）。
- 官方文档强烈推荐阅读：https://alembic.sqlalchemy.org/en/latest/tutorial.html。

Alembic 是 SQLAlchemy 生态的标准工具，掌握后能极大提升数据库变更的管理效率。如果你在具体框架（如 FastAPI + SQLModel）中使用，可以提供更多项目细节，我可以给出针对性配置示例。
