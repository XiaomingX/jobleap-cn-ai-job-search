# fly.io 是一个什么站点，它与 vercel.com有啥区别

⏺ Fly.io vs Vercel

  Fly.io 是一个 容器化应用部署平台，专注于在全球边缘节点运行 Docker 容器和虚拟机。它让你可以在靠近用户的地理位置部署任意语言的应用程序。

  主要区别：

  ┌──────────┬───────────────────────────────────┬────────────────────────────────────┐
  │   维度   │              Fly.io               │               Vercel               │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 核心定位 │ 通用容器/VM 部署平台              │ 前端/Jamstack 专用平台             │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 支持语言 │ 任意（Go, Rust, Python, Node 等） │ 主要针对 Next.js/React 生态        │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 部署方式 │ Docker 容器、VM                   │ Git push、Serverless Functions     │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 适用场景 │ 全栈应用、数据库、有状态服务      │ 前端网站、静态站点、API Routes     │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 数据库   │ 可直接部署 Postgres、Redis 等     │ 需外接（Supabase、PlanetScale 等） │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 复杂度   │ 较高，需了解容器和基础设施        │ 低，零配置部署                     │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 成本模型 │ 按资源（CPU/内存/存储）计费       │ 按请求量和带宽计费                 │
  ├──────────┼───────────────────────────────────┼────────────────────────────────────┤
  │ 边缘能力 │ 全球 30+ 区域，完整的边缘计算     │ Edge Functions（有限制）           │
  └──────────┴───────────────────────────────────┴────────────────────────────────────┘

  选择建议：

  - 选 Vercel：Next.js 项目、纯前端、需要快速上线、不想管基础设施
  - 选 Fly.io：需要运行自定义服务、有状态应用、数据库、WebSocket、需要更灵活的部署控制

  你当前的 FastAPI 项目如果要部署，Fly.io 更合适，因为 Vercel 的 Serverless Functions 对 Python 后端支持有限且有执行时间限制。
