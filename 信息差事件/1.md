**阿里（钉钉）的悟空平台（Wukong）目前主要面向企业级AI Agent/技能开发，没有提供独立的官方Python SDK供外部直接调用API。** 它更强调**Skill（技能）生态**和集成方式，支持开发者通过多种方式扩展能力。

### 1. **悟空平台的核心开发方式（非传统SDK）**
- **Skill开发**：悟空的核心扩展机制。你可以用Markdown定义Skill（描述意图、步骤、工作流），然后上传到悟空平台。Skill可以调用内置工具、MCP（Model Context Protocol）、RPA自动化流程等。
  - 支持**Python脚本执行**（平台内置Python 3.12运行时 + uv包管理器），适合本地代码执行、自动化任务。
  - 示例：通过RPA开发自动化流程 → 发布为MCP Tool → 在悟空Skill中调用。
- **文档与入口**：悟空官网（wukong.dingtalk.com）有Skill上传、MCP配置等指南。目前处于邀测/早期阶段，开发者主要通过平台UI和Markdown配置。

### 2. **可配合使用的Python相关工具/SDK**
悟空深度集成**钉钉开放平台**，你可以结合以下Python资源开发Skill或集成：

- **钉钉Python SDK**（推荐）：
  - 官方/社区版支持企业内部开发、机器人、Stream模式事件推送等。
  - 安装示例：`pip install dingtalk-stream`（Stream模式）或其他dingtalk-sdk包。
  - 用于：调用钉钉API、事件处理、与悟空联动（悟空可原生操作钉钉能力）。

- **阿里云相关Python SDK**（间接支持）：
  - **Alibaba Cloud Python SDK V2**（推荐）：`pip install alibabacloud` 系列，支持阿里云服务（如RPA、OSS、PAI等），悟空Skill常结合这些后端能力。
  - **PAI Python SDK**：用于AI模型训练/部署。
  - **RPA相关**：用于开发自动化Skill，Python可调用RPA API。

- **其他**：
  - 通义千问（Qwen）相关SDK：悟空内置千问模型，可通过阿里云模型API的Python客户端调用。
  - 平台内置Python环境：Skill中可直接运行Python代码，无需额外SDK。

### 3. **注意事项**
- 悟空是**企业级Agent平台**（2026年3月发布），重点在**Skill市场 + 阿里生态集成**（淘宝、支付宝、阿里云等能力以Skill形式接入），而非提供通用REST SDK。
- 开发者目前以**Skill + MCP + RPA**为主。如果需要底层API集成，优先用**钉钉开放平台SDK**。
