# 后端技术面试百科 (MySQL, Redis, Nginx, OS, Network)

## 一、MySQL 核心面试题 (Top 10)

1. **索引失效场景**：
   - 使用 `OR` 连接且非索引列；
   - 字符串未加引号导致类型转换；
   - `LIKE` 以 `%` 开头；
   - 联合索引不符合最左匹配原则；
   - 对索引列使用函数或运算。
2. **InnoDB 与 MyISAM 区别**：
   - InnoDB 支持事务、外键、行级锁、MVCC；MyISAM 仅支持表级锁，查询 `count(*)` 较快。
3. **B+ 树优势**：
   - 非叶子节点不存数据，降低树高，减少磁盘 I/O；叶子节点双向链表，适合范围查询。
4. **事务隔离级别**：
   - 读未提交、读已提交、可重复读 (MySQL 默认)、可串行化。
5. **死锁排查**：
   - `show engine innodb status` 查看日志，分析 SQL 加锁情况，约定相同访问顺序。

## 二、Redis 高频考点

1. **数据类型**：String, Hash, List, Set, ZSet。
2. **持久化机制**：RDB (快照)、AOF (日志)。推荐混合持久化。
3. **缓存异常**：
   - **穿透**：查询不存在的数据。解法：布隆过滤器、空值缓存。
   - **击穿**：热点 Key 过期。解法：永不过期、互斥锁。
   - **雪崩**：大量 Key 同时过期。解法：随机 TTL、分布式部署。
4. **主从同步**：全量同步 (RDB) 与增量同步 (指令流)。

## 三、网络与操作系统

1. **TCP 三次握手/四次挥手**：SYN-ACK 确认机制，TIME_WAIT 状态的作用。
2. **HTTP vs HTTPS**：HTTPS 增加了 SSL/TLS 层，非对称加密交换密钥，对称加密传输数据。
3. **进程与线程**：进程是资源分配单位，线程是调度单位。
4. **虚拟内存**：分页机制，LRU 置换算法。

## 四、Nginx 实战

1. **反向代理与负载均衡**：轮询、权重、IP Hash。
2. **常用指令**：`location`, `rewrite`, `proxy_pass`。
